# デブサミ2026 Summer ブース運用ボード

イベントブース運営のための、スマホで見るリアルタイム進行ボード。
**公開URL: https://casa2024takayama.github.io/devsumi-booth-board/**

- 対象イベント：Developers Summit 2026 Summer（2026年7月16日（木）・17日（金）／JPタワー ホール＆カンファレンス）
- 体制：両日とも4名（役割①〜④）／**ブースに立てるのは最大4名**
- シフト設計の詳細は [devsumi-summer-2026-booth-shift.md](devsumi-summer-2026-booth-shift.md)

---

## これは何をするもの？

「いま会場がどう動いていて、自分はあと何分で休憩か」を、当日ブースに立つ全員がスマホで一目で確認するためのボードです。3つの視点を1画面に持っています。

| 視点 | 場所 | 見えるもの |
|---|---|---|
| 全体の流れ | 進行表（メイン） | タイムテーブル・混雑レベル・NOWライン |
| いまの体制 | 「ただいまの混雑」パネル | ブースに立つ人／スタンバイ／休憩中と、それぞれの残り時間 |
| 自分の残り時間 | 担当者モード（時計の下） | 対応中なら次の休憩まで、休憩中なら戻りまでの分数 |

## 主な機能

### 進行表とNOWライン
- 現在時刻の赤いラインが進行表の上を移動し、過ぎた行は自動でグレーアウト
- 混雑レベル5段階（設営撤収／最閑散／閑散／混雑／ピーク）をリボンの太さと色で表示
- 日次タブで7/16・7/17を切り替え

### 担当者モード（「わたし」チップ）
- 時計の下の①〜⑤から自分の番号をタップすると、自分視点の残り時間が常時表示
- 対応中：「次の休憩まであと◯分（何時から何分）」
- 休憩中：「あと◯分（何時からブース脇スタンバイ）」
- 選択は端末ごとに保存。もう一度タップで解除

### スタンバイ制（このシフトの肝）
**各休憩の最後5分は「休憩」ではなく「ブース脇スタンバイ」。**
講演が予定より早く終わって人の波が前倒しになっても、脇に控えている人が即入りして実質5名で受けるための仕組み。残り5分になると：
- 担当者モードが警告色で「ブース脇へ移動。◯:◯◯交代」の行動指示に変わる
- メンバー一覧のバッジが「在席／休憩中／スタンバイ」の3状態で切り替わる
- 混雑パネルにスタンバイ中の人が独立した行で表示される

### メンバー登録
- 左パネル（スマホでは上部）で役割①〜⑤に名前を登録すると全表示に反映
- **名前は各端末のlocalStorageにのみ保存**（公開ページには焼き込まれない）
- 日ごとの稼働人数に対応（両日とも①〜④の4枠）
- 7/16→7/17のローテーション（木①→金②…、責任者は木④→金①）はデータに設定済み。名前は一度入れれば両日に正しく連動

### その他
- **時刻を仮設定**：任意の時刻にジャンプしてシフトを予習できる（前日ブリーフィングで15:25のピークを見せると効果的）
- **進行表を編集**：行の追加・削除、時刻・メモ・混雑・休憩者の変更
- **書き出す／読み込む**：全データをJSONでエクスポート・インポート
- **別イベントを作る**：進行表を複製して他イベントに流用

## シフト設計のルール（要点）

1. **休憩・交代の合図は時計ではなくセッション開始・転換開始**
2. **どの休憩も終わり5分前にブース脇スタンバイ**（波が早く来たら即入り）
3. **リフレッシュタイム（15:25–16:10）は全力体制**：木曜は4名全員・休憩なし、金曜はブース4名死守の15分交代（②→③→④）
- 両日とも「4-3-2」変則＋縁固め＝最閑散2名／閑散セッション3名／転換・ピーク・開場と各枠の縁（前後5〜20分）は4名
- 金曜はさらに14:55〜16:10の75分を休憩なしの全員在席で走り切る（B・C会場close→ピークの連続波）。詳細はシフト設計書を参照

## 当日の使い方（スタッフ向け）

1. 公開URLをスマホで開く（ホーム画面に追加推奨）
2. メンバー登録に名前が入っているか確認
3. 「わたし」チップで自分の番号をタップ（**7/17は役割が変わるので朝に選び直す**。木①→金②…、責任者は木④→金①）
4. あとは時計の下の自分の行だけ見ていればOK。スタンバイ表示が出たらブース脇へ

**1日目→2日目の反映**：当日の編集は各端末のlocalStorageに閉じるため、他の端末には自動反映されません。木曜夜にリーダーが金曜の行を修正 →「書き出す」でJSON保存 → チームに配布して各自「読み込む」（またはHTMLに焼き込んで再デプロイ）。振り返りの進め方はシフト設計書の第8章参照。

## 技術メモ

- 単一HTML（`index.html`）・依存ライブラリなし・ビルド不要
- データ保存は `localStorage`（キー：`booth-ops-v5`、担当者選択は `booth-ops-v5-me`）。**端末ごとに独立**で、端末間の同期はしない（進行データはHTMLに焼き込み済みなので、編集しない限り全端末同じ表示）
- シフトを変更する場合は `index.html` の `day1()` / `day2()` とシフト設計書mdの両方を更新し、push すると GitHub Pages が自動再デプロイ
- ホスティング：GitHub Pages（`main` ブランチ直下）
